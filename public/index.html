No. 1 · HN

I love the work of the ArchWiki maintainers

From linkThe author marks an appreciation day by thanking the ArchWiki maintainers, describing years of relying on the wiki’s clear, concise Linux documentation and noting how often it helps even outside Arch. The post urges readers to recognize the volunteer effort and support the project so the documentation stays sharp and current.

From commentsHN readers echoed that ArchWiki is a go-to reference even for non-Arch distros, compared it with Gentoo and other documentation ecosystems, and raised concerns about discoverability as search and AI indexing change, while celebrating the volunteer doc culture.

No. 2 · HN

Flashpoint Archive – Over 200k web games and animations preserved

From linkFlashpoint is a community preservation project that packages a launcher, proxy, and sandbox to keep legacy web games and animations playable across Flash and other plugins. It now catalogs over 200k titles, offering a lightweight “Infinity” install or the full “Ultimate” archive for collectors.

From commentsHN commenters swapped nostalgia and preservation tips, asked about missing titles and quality control, and discussed how tools like Ruffle fit into a broader strategy of emulation versus full archival collections.

No. 3 · HN

Two different tricks for fast LLM inference

From linkSean Goedecke contrasts two latency strategies: Anthropic keeps the model but sacrifices batching to respond faster, while OpenAI leans on Cerebras hardware plus a smaller model for speed. The post breaks down how memory bandwidth, batching, and hardware choices shape inference responsiveness.

From commentsHN discussion zoomed in on Cerebras capacity and model sizing, with some readers skeptical of the implied parameter counts and others dissecting memory-bandwidth bottlenecks, batching tradeoffs, and how fast-mode claims should be interpreted.

No. 4 · HN

Oat – Ultra-lightweight, semantic, zero-dependency HTML UI component library

From linkOat is a tiny, semantic HTML+CSS UI library that keeps dependencies at zero while offering clean, accessible components and sensible defaults. It focuses on minimal markup and easy customization so teams can ship lightweight UI without heavy frameworks.

From commentsCommenters debated component choices like the date picker and accordion behavior, compared Oat with classless kits such as Pico or Bootstrap, and flagged some components as feeling sluggish in the demo build.

No. 5 · HN

My smart sleep mask broadcasts users' brainwaves to an open MQTT broker

From linkThe author reverse-engineers a smart sleep mask by sniffing BLE traffic, decompiling the Flutter app, and uncovering shared MQTT credentials. They show how EEG data and device commands are exposed through a public broker, highlighting the privacy risk of insecure IoT backends.

From commentsHN readers zeroed in on the privacy implications, arguing for local-only operation and better credential handling, and treated it as another cautionary example of IoT security debt baked into consumer hardware.

No. 6 · HN

I Fixed Windows Native Development

From linkThe author documents the pain of Visual Studio’s GUI-driven installs and introduces msvcup, a scriptable installer that fetches specific MSVC and SDK versions for reproducible native builds. It aims to make Windows toolchains installable like other package-managed dependencies.

From commentsHN commenters suggested LTSC Build Tools and existing MSVC packages, debated licensing and distribution constraints, and split between “the tooling is already fine” and “this finally makes Windows builds scriptable.”
